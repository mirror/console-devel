Changelog

2.0 148c (16/12/2012)

  • Like in most code editors (Visual Studio, Notepad++, UltraEdit, etc.) the Ctrl-TAB switch the tabs in the order they have been visited instead of in the order they are displayed.

2.0 148b (8/10/2012)

  • Introduce a new command 'Interupt' that the user can associate to a short cut (e.g.: Ctrl+I) to replace Ctrl+C that is usually associated to the Copy selection command.
  • Introduce the -p flag that allow to post a text to console. This is a first step toward the replacement of Cmd.exe by Console.exe to manage the .bat, .cmd files.
  • Fix a bug in the Extplorer integration dll that prevent the explorer integration to introduce more than one sub menu in the explorer context menu (e.g.: Run Console and Run Console Console Tab).

2.0 148a (6/9/2012)

  • Make sure that commands that contains path with space are passed correctly to an external Console.exe when it run in one instance mode.

2.0 148 (4/2/2012)

Most changes in this build were done by Kirill. Thanks man!

  • Add configuration setting to allow cmd-style copy/clear selection click
  • Allow double-click to select a word (text between two spaces)
  • Allow new selection without clearing the old selection
  • Prevent pasting by mouse if pasting by menu is disabled
  • Limited max window transparency to 250 (i.e. min alpha is 5)
